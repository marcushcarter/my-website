How to Add SL_Image to your project

The first step to rendering textures in SDL is to add SDL_image to our repository. Visit the SDL_image GitHub page and download the file called SDL3_image-devel-3.2.4-mingw.tar.gz. Save it into the root of your repository and extract the file there.

Do you remember how to check what type of system your computer uses (either 32 or 64-bit)? If you are not sure go back and learn how to check your system type. We will again have to use the matching SDL_image folder for it to work:

  • i686-w64-mingw32 for 32-bit systems
  • x86_64-w64-mingw32 for 64-bit systems

We are only interested in three folders.

  • Copy the include folder into your project root.
  • Copy the lib folder into your project root.
  • Inside the bin folder, copy the SDL3_image.dll file into your project root.

Now we have to link the SDL_image DLL to our Makefile so it builds with it. At the end of the build command on our Makefile, add a new link to -lSDL3_image, and finally in the main.c file add the include #include <SDL3_image/SDL_image.h> under the SDL.h include. Notice how linking SDL_image was excactly the same process as linking SDL in the last lesson. This is how you link external header from any other place you want.

You can now delete all the other draw functions we used in the last lesson that are in the game loop (the while loop in the main function) and we'll start fresh. After you have done this, you can delete the original tar.gz file and the extracted folder.

	}

}

How to Render a Texture

The first thing we need to do when rendering a texture is to actually load it in. To do this we will make our own function. Above your main function make a new bool function called loadTextures() with an argument for a pointer to a SDL_Renderer object.

In this function, we will load individual textures to thier own texture objects. FOr eacah texture you want to load, you will add a new SDL_Texture* object. Mine will be called my_texture. Then, in our new Load Textures function we will attach our image to that object. We will do this with my_texture = IMG_LoadTexture(renderer, "./resources/shrek.png");

Your load textures function should now look something like this:


// Under the header files with the other objects
SDL_Texture* my_texture;

// Somewhere above the main function
bool load_textures(SDL_Renderer* renderer) {

	my_texture = IMG_LoadTexture(renderer, "./resources/shrek.png");
	// SDL_SetTextureScaleMode(texture, SDL_SCALEMODE_NEAREST);

	return true;
}
  1. add SDL_Texture* texture-name and SDL_Surface *textureSurface objects
  2. make a new load_textures function with renderer argument
  3. add (texture = IMG_LoadTexture(renderer, "./resources/shrek.png");) and (SDL_SetTextureScaleMode(texture, SDL_SCALEMODE_NEAREST);) inside this, finish it off with a return 0;
  4. make the texture postion object in the main unction (SDL_FRect rposition = {0, 0, 800, 600};)
  5. Render the texture (SDL_RenderTexture(renderer, texture, NULL, &rposition);)
